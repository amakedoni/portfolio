Как адаптировать сайт под мобильные?

Выбери подход — от него зависит объём работы и результат

A

Аудит и точечные исправления

Пройтись по сайту на 375px, найти конкретные поломки (переполнение текста, невидимые элементы, мелкие кнопки) и починить только их в существующем style.css.


Плюсы: Быстро, минимальный риск что-то сломать на десктопе

Минусы: Проблемы могут вернуться при добавлении нового контента

Когда выбрать: Если сайт уже почти работает, нужен быстрый результат

B

Отдельный файл mobile.css

Создать новый mobile.css со всеми мобильными оверрайдами. Подключить его через <link media="(max-width: 768px)">. Основной CSS не трогаем.


Плюсы: Изолировано, безопасно, легко откатить

Минусы: Два файла — дублирование переменных, сложнее поддерживать

Когда выбрать: Если боишься что-то сломать в десктопной версии

C

Полный mobile-first рефакторинг

Переписать style.css с нуля по принципу «сначала мобильные стили, потом desktop через min-width». Стандартный современный подход.


Плюсы: Правильная архитектура, хорошо масштабируется

Минусы: Большой объём работы, риск регрессий на десктопе

Когда выбрать: Если готов к серьёзному рефакторингу
